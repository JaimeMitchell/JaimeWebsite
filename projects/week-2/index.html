Git and Github had been something I've had a 10 minute experience. As part of my Corona lock down I bought a Raspberry Pi and Elegoo set (Arduino kit knockoff) I joined a free online class (CS50) to learn more about computer science in order to understand embedded coding a bit more.

The class required Github to turn in assignments but I still didn't really get what git was for. Not until last week, when the desktop I was using died on me during my website project. The one time in 15 years I didn't back up my work...

What I learned in the process:

  • When the desktop died the Raspberry pi took over. I found a knockoff version of Visual Studio Code for the Pi here.

  • A repository is the folder of my website's code

  • Git automatically takes snapshots of everything I do to my repositoy, like a spy tracking my every move. Or like history in Adobe Lightroom.

  • Git needs to be spoken to in the Command Line. The tone should be direct, sequential and simple as if speaking to a puppy. "git add. git commit -m ("comment"). git push origin master..."

  • I have to have my files in the correct location (the Pi Home folder for me) then initialize Git to that folder. If Git can't see the file clearly it will ignore me and tell me I'm making fatal errors which felt a little passive aggresive...
